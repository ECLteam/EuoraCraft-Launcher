高危问题

19 项
#1
launch_instance 账户字段名错误 — 游戏始终以 "Player" 启动
高危后端逻辑错误
ECL/api/handlers.py:1058-1061
get_current_account() 返回的字典键为 "alias"，但 launch_instance 使用 current.get("name", "Player")，导致玩家名始终为 "Player"。同理 access_token 键不存在，微软账户令牌始终为字符串 "None"，微软账户无法正常认证启动。
修复：改为 current.get("alias", "Player")，通过 account_manager 获取令牌。
#2
AppState.initialize() 返回类型不一致 — 配置校验失败时崩溃
高危后端运行时Bug
ECL/common/state.py:104-113
initialize() 声明返回 dict[str, Any]，但配置校验失败时返回 False (bool)。调用方 launcher.py:188 执行 state_result.get("needs_password") 会抛 AttributeError: 'bool' object has no attribute 'get'。
修复：返回 {"needs_password": False, "error": error} 形式的字典。
#3
CustomFallbackKeyring 每次生成新密钥 — 密码数据永久丢失
高危后端数据丢失
ECL/auth/crypto.py:93-98
__init__ 每次调用 Fernet.generate_key() 生成新密钥但不持久化。每次重启后新密钥无法解密旧数据，get_password 中解密异常被 except Exception: continue 静默吞掉（132行），所有已存储密码永久丢失。
修复：将密钥持久化到文件中，初始化时加载已有密钥。
#4
_safe_callback 超时机制失效 — shutdown 死锁
高危后端线程安全
ECL/plugin/framework.py:671-680
使用 with ThreadPoolExecutor 上下文管理器，future.result(timeout) 超时后返回 None，但 with 退出时调用 pool.shutdown(wait=True) 永久阻塞等待挂起的插件回调。如果插件 on_disable/on_unload 卡住，整个 shutdown 流程死锁。
修复：改用 pool.shutdown(wait=False) 或不使用上下文管理器。
#5
Zip Slip 路径遍历漏洞 — 插件安装
高危后端安全隐患
ECL/api/handlers.py:1283-1284
plugin_install 中 zf.extractall(target_dir) 直接解压 ZIP，未校验压缩包内文件路径。恶意 ZIP 可包含 ../../ 路径遍历条目，将文件写到目标目录之外。
修复：解压前检查每个条目的规范化路径是否在 target_dir 内。
#6
_ensure_initialized() 返回类型不一致 — 初始化失败后继续执行
高危后端逻辑错误
ECL/auth/manager.py:74-77
已初始化时返回 True (bool)，未初始化时返回 self.initialize() 的结果 (dict)。非空字典始终为 truthy，not {"success": False} = False，初始化失败时不会提前返回，后续在 _auth 未完全初始化时抛 RuntimeError。
修复：统一返回 bool 类型。
#7
api_call 异常处理过窄 — 未捕获异常传播到前端
高危后端异常处理
ECL/adapters/adapter.py:59-65
api_call 中 func(*args) 仅捕获 TypeError，但 Api 方法可能抛出 RuntimeError、OSError、ValueError 等，这些异常直接传播到 Tauri 命令处理器，导致前端收到不可预期的错误。
修复：扩大捕获范围或确保所有 Api 方法都有完善的异常处理。
#8
logger 未定义 — InstancesManager 清理时 NameError 崩溃
高危核心运行时Bug
ECL/game/Core/InstancesManager.py:168
shutdown_all 中使用 logger.warning()，但文件顶部从未导入 logger。当 proc.wait(timeout=5) 超时触发 TimeoutExpired 时，抛出 NameError: name 'logger' is not defined，导致 shutdown_all 崩溃且无法继续清理其他实例。
修复：添加 from ECL.common.logger import get_logger; logger = get_logger("instances")
#9
name_to_path 返回 None 导致 TypeError — 启动崩溃
高危核心运行时Bug
ECL/game/Core/ECLauncherCore.py:176, 224

修复：添加 if not get_path: continue 检查。
#10
